# Django 5.2 LTS 업그레이드

**날짜**: 2026-03-10

## 작업 내용

Django 5.1.15 → 5.2.12 LTS 업그레이드

## 이유

- Django 5.1은 2025년 12월 보안 지원 종료
- Django 5.2는 LTS로 2028년 4월까지 보안 지원

## 진행 과정

1. `pip install Django==5.2.12` 설치
2. `python manage.py check` — fsis/ghdb 모두 이슈 없음
3. 서드파티 패키지 호환성 확인 (simple_history, import_export, autocomplete-light, bootstrap5, multiselectfield, imagekit)
4. 미적용 마이그레이션 3건 적용
   - `fsis.0004_historicalmuseumspecimen_rock_category_and_more`
   - `fsis.0005_historicalmuseumspecimen_source_system_and_more`
   - `kprdb.0002_historicalreference_has_text_layer_and_more`
5. 개발 서버 정상 실행 확인
6. `requirements.txt` 업데이트
7. Docker 버전 0.2.4 → 0.2.5

## 변경 파일

- `requirements.txt` — Django==5.2.12
- `deploy/DOCKER_VERSION` — 0.2.5
