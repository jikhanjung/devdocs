# Docker 이미지 크기 문제 해결 (0.2.3)

**날짜**: 2026-03-08

## 문제

- Docker 이미지 크기가 0.2.0(474MB)에서 0.2.1/0.2.2(1.56GB)로 급증
- 개발서버가 아닌 운영서버에서 이미지를 빌드하면서 발생

## 원인

운영서버에 존재하는 테스트 데이터 파일들이 `.dockerignore`에 누락되어 이미지에 포함됨:

- `testdata/` 디렉토리 (264MB) — 테스트용 업로드 파일 및 fixture
- `testdata.tar.gz` (257MB)
- 합계 약 521MB 추가

## 조치

1. `.dockerignore`에 `testdata/` 추가
2. `testdata.tar.gz` 파일 삭제
3. `testdata/` 디렉토리 삭제
4. 0.2.3 빌드 및 push → **473MB** (0.2.0 수준으로 복원)
5. 기존 0.2.0, 0.2.1, 0.2.2 이미지 삭제
6. fsis, ghdb 컨테이너 0.2.3(latest)으로 재시작

## 결과

| 버전 | 이미지 크기 |
|------|------------|
| 0.2.0 | 474MB |
| 0.2.1 | 1.56GB |
| 0.2.2 | 1.56GB |
| **0.2.3** | **473MB** |

디스크 사용량: 48GB 중 37GB 사용 (78%, 11GB 여유)
