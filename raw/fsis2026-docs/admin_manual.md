## 1. 관리자 역할 개요

관리자는 일반사용자가 입력한 논문/표본 데이터를 검토하고, 화석산지 등록 및 위험도 평가까지의 전체 워크플로우를 관리합니다.

### 1.1 작업 흐름

```
일반사용자 입력                     관리자 작업
──────────────────                ──────────────────────────────────
논문 등록 (PDF 업로드)     →      PDF 처리 파이프라인 실행
분류군/표본 정보 입력      →      입력 데이터 검토/확인
표본 위경도 입력           →      추출 좌표 검토 (검토/관리필요/제외)
                                  ↓
                                  화석산지 후보 지도에서 산지 등록
                                  ↓
                                  화석산지 위험도 평가
```

### 1.2 관리자 메뉴

로그인 후 상단 네비게이션의 **시스템관리** 드롭다운에서 접근합니다:

| 메뉴 | URL | 설명 |
|------|-----|------|
| 사용자 목록 | `/kprdb/user_list_admin/` | 사용자 계정 관리 |
| FSIS 접속기록 | `/fsis/all_activity_list_admin/` | FSIS 사용 기록 |
| KPRDB 접속기록 | `/kprdb/all_activity_list_admin/` | KPRDB 사용 기록 |
| 편집기록 | `/kprdb/history_overview/` | 최근 편집 이력 |
| PDF 처리현황 | `/kprdb/pdf_status/` | 텍스트 레이어/OCR/추출 현황 |
| 추출 표본번호 | `/kprdb/extracted_specimens/` | PDF에서 자동 추출된 표본번호 |
| KPRDB 표본목록 | `/kprdb/taxon_specimens/` | 수동 입력 표본과 FSIS 매칭 |
| 화석산지 후보 지도 | `/kprdb/site_candidates/` | 좌표 aggregate 지도 + 산지 등록 |

## 2. 입력 데이터 검토

### 2.1 편집기록 확인

1. **시스템관리** > **편집기록** 접속
2. 최근 편집된 논문, 저널, 저자, 분류군의 변경 이력 확인
3. 각 항목의 변경 전/후 값을 비교하여 오류 확인

### 2.2 논문 데이터 검토

1. **KPRDB** > **논문목록**에서 최근 등록된 논문 확인
2. 논문 상세 페이지에서 확인할 항목:
   - 기본정보 (제목, 연도, 저널, 저자) 정확성
   - 분류유형 적절성 (분류학/비분류학/해외화석)
   - 분류군(Taxa) 정보 — 학명, 지층, 시대 일치 여부
   - 표본 정보 — 표본번호 형식, 위경도 정확성

### 2.3 표본 매칭 확인

두 가지 표본 매칭 페이지를 활용합니다:

**KPRDB 표본목록** (`/kprdb/taxon_specimens/`)
- 사용자가 수동 입력한 ReferenceTaxonSpecimen 467건
- FSIS MuseumSpecimen과 표본번호 기준 매칭
- [FSIS 매칭] 배지 클릭으로 매칭된 건만 필터링

**추출 표본번호** (`/kprdb/extracted_specimens/`)
- PDF에서 자동 추출된 표본번호 664건
- 기관별 필터 (KOPRIF, JBNU, SNU 등)
- FSIS/KPRDB 양쪽 매칭 결과 표시

## 3. PDF 처리 파이프라인

### 3.1 처리 현황 확인

1. **시스템관리** > **PDF 처리현황** 접속
2. 상단 통계 카드에서 전체 현황 확인:
   - **텍스트 레이어**: 있음/없음/미확인
   - **OCR**: 완료/실패/미처리
   - **UID**: 있음/없음
   - **산지 추출**: 완료/미시도/데이터없음
3. 배지 클릭으로 해당 상태의 논문만 필터링

### 3.2 개별 논문 처리

논문 상세 페이지 하단의 **논문 처리** 섹션에서:

1. **텍스트 레이어 확인** — [▶] 버튼 클릭
2. **OCR 처리** — 스캔본인 경우 활성화, [▶] 클릭 (수 분 소요)
3. **UID 생성** — [▶] 클릭
4. **산지/표본 추출** — [▶] 클릭

또는 **[전체 실행]** 버튼으로 1~4단계 순차 실행.

### 3.3 일괄 처리 (서버 커맨드)

대량 처리가 필요한 경우 서버에서 management command를 실행합니다:

```bash
# venv 활성화 + 환경변수
export DATABASE_PATH=/srv/fsis2026/db.sqlite3
export MEDIA_ROOT=/srv/fsis2026/uploads
PYTHON=/home/honestjung/venv/fsis2026/bin/python

# 1. 텍스트 레이어 일괄 확인
$PYTHON manage.py check_text_layers

# 2. OCR 일괄 실행 (스캔본 대상, 건당 5~15분)
$PYTHON manage.py run_ocr

# 3. UID 일괄 생성
$PYTHON manage.py generate_uids

# 4. 좌표/표본번호 일괄 추출
$PYTHON manage.py extract_pdf_data --all
```

> **참고**: 호스트에서 management command 실행 후 WAL 체크포인트가 자동 적용됩니다.
> Docker 컨테이너에서 변경사항이 즉시 보이지 않으면 `docker compose restart`로 해결됩니다.

### 3.4 OCR 실패 처리

- PDF 처리현황에서 OCR 실패 배지 클릭 → 실패 건 목록
- 개별 논문 상세에서 재시도 가능
- 파일 손상이나 특수 형식의 경우 수동 처리 필요

## 4. 추출 좌표 검토

PDF에서 자동 추출된 좌표는 반드시 사람의 검토를 거쳐야 합니다.

### 4.1 좌표 상태 분류

논문 상세 페이지의 **PDF 추출 데이터** > **좌표** 테이블에서 각 좌표의 상태를 라디오버튼으로 선택합니다:

| 상태 | 의미 | 후속 작업 |
|------|------|-----------|
| **검토** | 아직 확인하지 않음 (기본값) | 확인 필요 |
| **관리필요** | 화석산지로 등록/관리 대상 | 화석산지 후보 지도에서 등록 |
| **제외** | 화석산지가 아님 (해양 좌표, 비화석 산지 등) | 후보 지도에서 자동 제외 |
| **등록됨** | 이미 FossilSite로 등록 완료 | 추가 작업 없음 |

### 4.2 검토 기준

좌표를 **제외**로 분류하는 경우:
- 논문의 연구 지역 개관에서 나온 일반적 지명 좌표
- 해외 비교 표본의 좌표
- 지질도/지형도 참조 좌표 (화석 산출과 무관)

좌표를 **관리필요**로 분류하는 경우:
- 화석이 실제로 산출된 지점의 좌표
- 논문에서 locality/section으로 명시한 좌표
- 표본의 산출지 좌표

### 4.3 상태 변경

- 라디오버튼 선택 → 즉시 자동 저장 (행이 잠시 노란색 → 녹색 깜빡임)
- 변경 이력은 논문의 `extracted_data` JSON에 기록됨

## 5. 화석산지 관리

### 5.1 화석산지 후보 지도

1. **시스템관리** > **화석산지 후보 지도** 접속
2. 지도에 세 종류의 마커가 표시됩니다:

| 색상 | 의미 |
|------|------|
| **녹색** | 기존 등록된 FossilSite |
| **파란색** | 좌표 그룹이 기존 산지에 연결됨 |
| **주황색** | 미등록 후보 — 검토 후 등록 필요 |

3. 좌측 사이드바에서:
   - **후보만** / **연결됨** / **전체** 필터 선택
   - 검색으로 특정 위치/논문 찾기
   - 항목 클릭 → 지도에서 해당 위치로 이동

### 5.2 후보에서 화석산지 등록

1. 주황색 마커 클릭 → 팝업 표시
2. 위치, 좌표, 관련 논문 정보 확인
3. **[화석산지 등록]** 버튼 클릭
4. 확인 대화상자에서 **확인** 클릭
5. 시스템이 자동으로:
   - FossilSite 생성 (site_code 자동 채번: `FS-YYYY-NNNN`)
   - Nominatim 역지오코딩으로 시도/시군구/읍면동 자동 입력
   - 관련 논문과 FossilSiteReference 연결
6. 등록 완료 후 버튼이 녹색으로 변경 + 산지코드 표시

### 5.3 화석산지 상세 편집

등록 후 추가 정보를 입력합니다:

1. **화석산지** > **산지목록**에서 해당 산지 클릭
2. **[편집]** 클릭
3. 추가 입력 항목:

| 필드 | 설명 |
|------|------|
| 산지명(영문) | 영문 지명 |
| 상세주소 | 정확한 주소 |
| 지질시대 | 드롭다운 선택 |
| 암석단위 | 지층명 선택 |
| 암종 | 암석 종류 |
| 화석군 | 주요 화석 종류 |
| 화석설명 | 산출 화석 상세 기술 |
| 학술적 가치 | 학술적 의의 |
| 보호현황 | 천연기념물/문화재자료/무보호 등 |
| 최초보고연도 | 논문 발표 연도 |

4. **[Save]** 클릭

### 5.4 화석산지-논문 연결 확인

- 화석산지 상세 페이지에 **관련 논문** 테이블이 표시됩니다
- 논문 상세 페이지에도 **관련 화석산지** 테이블이 표시됩니다
- 양방향 링크로 상호 참조 가능

### 5.5 화석산지 지도

**화석산지** > **산지지도**에서 전체 산지를 지도에서 확인할 수 있습니다:
- 마커 색상: 위험도 등급별 (P0 빨강 ~ P4 파랑, 미평가 회색)
- 좌측 필터: 시도, 보호현황, 우선순위
- 마커 hover/클릭 시 팝업 (산지명, 보호현황, 위험도, 상세보기 링크)

## 6. 위험도 평가

### 6.1 평가 대시보드

**화석산지** > **위험도 대시보드**에서 전체 평가 현황을 확인합니다.

### 6.2 새 평가 작성

1. 화석산지 상세 페이지에서 **[새 평가]** 클릭
2. Brilha 5-criteria 위험도 평가 항목 입력:

| 항목 | 설명 |
|------|------|
| 자연재해 위험도 | 침식, 붕괴, 풍화 등 |
| 인간활동 위험도 | 개발, 채굴, 채집 등 |
| 법적 보호 수준 | 천연기념물, 무보호 등 |
| 접근성 | 도로 거리, 접근 난이도 |
| 토지피복 | 주변 토지 이용 현황 |

3. 시스템이 자동으로:
   - Overpass API로 최근접 도로 거리 측정
   - 위성영상 기반 토지피복 분석
4. 종합 위험도 점수 및 우선순위 (P0~P4) 자동 산출

### 6.3 평가 이력

- 화석산지 상세에서 **[평가이력]** 클릭
- 동일 산지에 대한 복수 평가 시계열 확인 가능

### 7.1 사용자 관리

- **KPRDB 접속기록**: 논문/저자/저널 관련 접속 기록

- OCR 파일: 원본 옆에 `_ocr.pdf` 별도 보관

## 8. 서버 운영

### 8.1 배포

```bash
# 1. config/version.py 버전 수정
# 2. 빌드 및 push
docker build -f deploy/Dockerfile -t honestjung/fsis2026:X.Y.Z -t honestjung/fsis2026:latest .
docker push honestjung/fsis2026:X.Y.Z && docker push honestjung/fsis2026:latest

# 3. 프로덕션 배포
/srv/fsis2026/deploy.sh X.Y.Z   # fsis
/srv/ghdb/deploy.sh X.Y.Z       # ghdb

# 또는 PC에서 원격 배포
ssh dolfinid-deploy "/srv/fsis2026/deploy.sh X.Y.Z"
```

### 8.2 컨테이너 상태 확인

```bash
docker ps                          # 컨테이너 상태
docker logs fsis --tail 20         # fsis 로그
docker logs ghdb --tail 20         # ghdb 로그
```

### 8.3 Nginx

```bash
sudo systemctl status nginx
sudo systemctl reload nginx        # 설정 변경 후 재적용
```

포트 구조:
- `:443` (HTTPS) → `localhost:8003` → fsis 컨테이너 `:8000`
- `:8001` (HTTPS) → `localhost:8002` → ghdb 컨테이너 `:8000`

### 8.4 SQLite WAL 주의사항

호스트에서 management command를 실행하면 변경사항이 WAL 파일에 기록됩니다.
management command에 WAL 체크포인트가 내장되어 있으나, 수동으로 체크포인트가 필요한 경우:

```bash
python -c "
import sqlite3
conn = sqlite3.connect('/srv/fsis2026/db.sqlite3')
conn.execute('PRAGMA wal_checkpoint(TRUNCATE)')
conn.close()
"
```

### 8.5 OCR 환경 (호스트 전용)

OCR은 Docker 이미지에 포함되지 않으며 호스트에서만 실행 가능합니다:

```bash
# 설치된 패키지
ocrmypdf --version           # PDF OCR 도구
tesseract --version          # OCR 엔진
# 언어팩: eng, kor, chi_tra (번체자)

# venv
source /home/honestjung/venv/fsis2026/bin/activate
```

### 8.6 환경 파일

| 파일 | 위치 | 용도 |
|------|------|------|
| `.env` | `/srv/fsis2026/`, `/srv/ghdb/` | Docker compose 변수 (IMAGE_TAG) |
| `.env.django` | `/srv/fsis2026/`, `/srv/ghdb/` | Django 환경변수 (SECRET_KEY, DEBUG 등) |
| `deploy.sh` | `/srv/fsis2026/`, `/srv/ghdb/` | 배포 스크립트 |
| `docker-compose.yml` | `/srv/fsis2026/`, `/srv/ghdb/` | 서비스별 compose 설정 |
