# PaperMeister 개인용 vs 연구실용 요구사항 정리

**날짜:** 2026-04-03  
**상태:** 초안  
**목적:** 개인용과 연구실용 사용 맥락의 차이, 그리고 연구실용으로 확장될 때 필요한 설계 관점을 정리

## 1. 문제의식

PaperMeister는 개인이 써도 유용할 수 있다.

하지만 연구실 단위로 올라가면 제품의 성격이 달라진다.

개인용에서는 주로:

- 흩어진 논문을 다시 찾기 쉽게 하는 것
- PDF를 읽을 수 있는 searchable corpus로 바꾸는 것

이 핵심이다.

반면 연구실용에서는:

- 여러 사람에게 흩어진 문헌을 하나의 자산으로 운영하는 것
- 개인에게만 있던 문맥과 경험을 공동 기억으로 전환하는 것

이 핵심이 된다.

즉, 개인용 PaperMeister는 문헌 검색 도구에 가깝고,  
연구실용 PaperMeister는 공동 기억과 문헌 continuity를 관리하는 지식 인프라에 가깝다.

## 2. 개인용과 연구실용의 본질적 차이

## 2.1 개인용의 중심 질문

- 내가 가진 논문을 잘 찾을 수 있는가
- 내 PDF가 검색 가능한가
- OCR과 검색 품질이 충분한가
- 기존 Zotero나 폴더와 잘 연결되는가

즉, 개인용에서는 효율과 편의가 우선이다.

## 2.2 연구실용의 중심 질문

- 연구실 전체 문헌 자산을 한데 모을 수 있는가
- 새 구성원이 기존 문헌 맥락을 물려받을 수 있는가
- 누가 어떤 자료를 추가했고, 무엇을 중요하게 봤는지 남는가
- 같은 논문이 여러 사람에게 흩어져 있을 때 어떻게 다룰 것인가
- 개인 자료와 공용 자료의 경계는 어떻게 관리할 것인가

즉, 연구실용에서는 운영과 거버넌스가 우선이다.

## 3. 개인용에서 중요한 요구사항

개인용에서는 아래 요구사항이 우선이다.

- 쉬운 import
- 강한 OCR
- 빠른 검색
- 기존 도구와의 충돌 없는 연동
- 로컬 우선 사용
- 최소 설정으로 동작

개인용에서는 provenance도 중요하지만, 연구실용만큼 엄격할 필요는 없다.

핵심은 “내가 가진 자료를 다시 쓸 수 있게 만드는 것”이다.

## 4. 연구실용에서 추가로 필요한 요구사항

연구실용으로 가면 개인용 요구사항 위에 다음이 추가된다.

- provenance 추적
- ownership 구분
- 공용/개인 경계 관리
- metadata trust 정책
- 중복/충돌 처리
- continuity 보장
- onboarding 용이성
- 보안과 접근 통제

즉, 연구실용은 단순히 “여러 사람이 쓰는 개인용 앱”이 아니다.

그보다 “여러 개인의 분절된 문헌 자산과 경험을 하나의 운영 가능한 지식 기반으로 바꾸는 시스템”에 가깝다.

## 5. 왜 연구실용이 더 어려운가

연구실에서는 문헌만 흩어져 있는 것이 아니다.

흩어져 있는 것은 다음과 같다.

- PDF 파일
- reference metadata
- 폴더 구조
- 찾는 방식
- 중요하다고 생각한 논문 목록
- 읽었던 맥락
- 누적된 경험

예를 들어 어떤 논문이 연구실 전체에서 중요한 자료라고 해도, 실제로는:

- 한 학생의 Zotero에만 있을 수 있고
- 다른 사람은 로컬 폴더에만 PDF를 갖고 있고
- 누군가는 그 논문의 중요성을 메모로만 알고 있고
- 새 구성원은 그 사실을 전혀 모를 수 있다

즉, 연구실 문제는 단순 저장 문제가 아니라  
**분절된 지식과 경험의 단절 문제**다.

PaperMeister의 잠재력은 바로 이 분절을 이어붙일 수 있다는 점에 있다.

## 6. 연구실용에서 꼭 보존해야 할 것

연구실용에서는 파일과 메타데이터만 보존해서는 충분하지 않다.

최소한 아래도 함께 보존해야 한다.

- 이 논문이 어디서 왔는가
- 누가 추가했는가
- 어떤 source와 연결되어 있는가
- 어떤 파일이 canonical인가
- 어떤 메타데이터를 신뢰하는가
- 연구실 공용 자산인지, 개인 자산인지

장기적으로는 다음도 중요해진다.

- 개인 노트와 공용 노트의 구분
- 개인 태그와 공용 태그의 구분
- 특정 논문이 연구실에서 왜 중요한지에 대한 맥락
- 어떤 주제/프로젝트와 연결되는지

즉, 연구실용 PaperMeister는 문헌 저장소를 넘어  
**문헌을 둘러싼 맥락까지 이어붙이는 기반**이 될 필요가 있다.

## 7. 개인용과 연구실용의 공통 기반

둘이 완전히 다른 제품이 되어야 하는 것은 아니다.

공통 기반은 충분히 공유할 수 있다.

- multi-source import
- OCR
- searchable corpus
- provenance 유지
- sync
- canonical paper/file 모델

즉, 코어 엔진은 공통이고, 차이는 운영 레이어에서 커진다.

## 8. 연구실용에서 추가될 운영 레이어

연구실용으로 갈 때는 다음 레이어가 중요해진다.

## 8.1 Provenance Layer

- source 정보
- 추가한 사람
- 파일 출처
- metadata 출처

## 8.2 Visibility Layer

- 개인 자료
- 공용 자료
- 제한 공유 자료

## 8.3 Trust Layer

- 어떤 metadata를 canonical로 볼지
- 충돌 시 무엇을 우선할지
- 수동 수정과 source sync 충돌을 어떻게 다룰지

## 8.4 Continuity Layer

- 새 연구실 구성원이 빠르게 문헌 기반에 적응할 수 있는가
- 졸업/퇴사 이후에도 문헌 자산과 맥락이 남는가

## 8.5 Knowledge Layer

- 구조화된 corpus
- 키워드/용어 분석
- 노트/태그/annotation
- 장기적으로는 assertion과 assistant

## 9. 제품적으로 중요한 함의

이 차이는 제품 전략에도 영향을 준다.

개인용에서는 가치가 이렇게 보인다.

- 내 PDF를 다시 찾게 해준다
- 스캔본까지 검색 가능하게 한다
- Zotero나 폴더를 버리지 않게 해준다

연구실용에서는 가치가 이렇게 보인다.

- 연구실 문헌 자산을 공동 기억으로 만든다
- 사람 교체에도 지식이 끊기지 않게 한다
- 흩어진 source를 운영 가능한 corpus로 바꾼다

즉, 연구실용의 판매 포인트는 검색 편의 그 자체보다  
**continuity와 shared knowledge infrastructure**에 가깝다.

## 10. 설계 원칙

앞으로 개인용과 연구실용을 함께 고려한다면 다음 원칙이 중요하다.

1. 코어 corpus 엔진은 공통으로 유지한다
2. provenance는 처음부터 보존한다
3. 개인 자료와 공용 자료를 구분할 수 있게 설계한다
4. canonical paper와 source record를 분리한다
5. annotation, tag, note는 장기적으로 개인/공용 이중 레이어를 고려한다

즉, 개인용을 만들더라도 연구실용으로 확장될 수 있는 방향이어야 한다.

## 11. 요약

PaperMeister의 개인용 가치는:

- 흩어진 논문을 다시 읽고 찾을 수 있게 만드는 것

PaperMeister의 연구실용 가치는:

- 분절된 문헌과 경험을 하나의 공동 기억과 corpus infrastructure로 바꾸는 것

그래서 연구실용으로 갈수록 중요한 것은 단순한 검색 기능보다:

- provenance
- ownership
- trust
- continuity
- shared knowledge

가 된다.

PaperMeister가 잘 되면, 단순한 논문 관리 도구를 넘어  
연구실의 분절된 지식과 경험을 이어붙이는 기반이 될 수 있다.
