# PaperMeister OCR 비용 최적화 계획

**날짜:** 2026-04-03  
**상태:** 초안  
**목적:** 개인 및 초기 파일럿 단계에서 OCR 비용을 통제하면서도 제품 유용성을 검증할 수 있는 운영 전략 정리

## 1. 문제

PaperMeister의 가치는 어느 정도 규모의 corpus가 실제로 OCR되어 searchable 상태가 되었을 때 비로소 체감된다.

즉, 몇 편의 PDF만 처리해서는 제품의 유용성을 평가하기 어렵다.

하지만 고품질 OCR을 대량으로 수행하면 비용이 빠르게 커진다.

따라서 핵심 질문은 다음이다.

**어떻게 하면 제품의 핵심 가치를 훼손하지 않으면서 OCR 비용을 효율적으로 통제할 수 있는가?**

## 2. 기본 인식

Chandra2의 가치는 단순한 텍스트 인식률만이 아니다.

특히 PaperMeister에서는 다음이 중요하다.

- 높은 OCR 정확도
- section/caption/body 구분
- figure/table 주변 정보 인식
- layout 구조 보존

즉, Chandra2는 단순 OCR 엔진이 아니라, **구조화 가능한 corpus를 만드는 엔진**으로 봐야 한다.

이 때문에 단순히 가장 싼 OCR 엔진으로 바꾸는 접근은 제품 방향과 잘 맞지 않을 수 있다.

## 3. 비용 최적화의 핵심 원칙

OCR 비용을 줄이는 가장 좋은 방법은:

**더 싼 OCR을 찾는 것보다, 고품질 OCR을 정말 필요한 PDF와 페이지에만 쓰는 것**

## 4. 권장 전략

## 4.1 OCR 필요 여부 판별

모든 PDF를 동일하게 OCR하지 않는다.

우선 다음을 먼저 판단한다.

- 텍스트 레이어가 충분히 존재하는가
- 텍스트 품질이 사용할 만한가
- OCR이 실제로 필요한가

가능한 기준:

- 추출 텍스트 길이
- 페이지당 텍스트 분포
- 문자 깨짐 정도
- 제목/초록 추출 가능 여부

이 기준을 통과하지 못하는 PDF만 고품질 OCR 대상으로 보낸다.

## 4.2 선택적 OCR

컬렉션 전체를 한 번에 완주하기보다 우선순위를 둔다.

예:

- 자주 참조하는 컬렉션 우선
- 현재 프로젝트 관련 논문 우선
- 스캔본과 OCR-poor PDF 우선
- 최근 추가된 논문 우선

즉, 비용을 전체 coverage보다 perceived usefulness 쪽에 먼저 배치한다.

## 4.3 페이지 단위 선택 처리

가능하다면 PDF 전체가 아니라 필요한 페이지만 OCR한다.

예:

- 텍스트가 거의 없는 페이지
- 앞부분 핵심 페이지
- 구조 추출 가치가 높은 페이지
- 문제 있는 페이지 재시도

이 전략은 구현 난이도는 있지만 장기적으로 비용 최적화 효과가 크다.

## 4.4 캐시 재사용 극대화

이미 있는 장점은 더 강화해야 한다.

- 파일 hash 기반 OCR 결과 재사용
- 같은 PDF가 여러 source에 있어도 한 번만 OCR
- 메타데이터만 바뀌면 OCR 재실행 금지
- raw OCR JSON 장기 보존

multi-source 구조로 갈수록 이 캐시는 더 중요해진다.

## 4.5 제품 검증용 subset 우선

제품 유용성 검증을 위해 반드시 전체 컬렉션 100% OCR이 필요한 것은 아니다.

더 현실적인 방식:

- 대표성이 있는 subset을 먼저 완성
- 자주 쓰는 분야나 프로젝트별 corpus를 먼저 구축
- 그 subset에서 검색/구조화/활용 가능성을 검증

즉, 초기에는 coverage보다 설득력 있는 경험을 우선한다.

## 5. 권장 실행 순서

1. 텍스트 레이어 품질 판별 도입
2. OCR necessity rule 정의
3. hash 캐시 재사용 강화
4. 컬렉션 우선순위 처리
5. 이후 필요 시 페이지 단위 selective OCR 도입

## 6. 요약

PaperMeister에서 OCR 비용 최적화는 저가 엔진 교체보다:

- OCR 필요 여부 판별
- 선택적 처리
- 캐시 재사용
- 대표 subset 우선 구축

이 더 중요하다.

즉, 고품질 OCR의 가치를 포기하지 않으면서도  
**필요한 곳에만 깊게 투자하는 방향**이 가장 현실적이다.
