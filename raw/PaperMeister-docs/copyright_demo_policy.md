# PaperMeister 저작권 및 데모 정책 메모

**날짜:** 2026-04-03  
**상태:** 초안  
**목적:** OCR 기반 corpus와 외부 데모 자산을 운영할 때의 기본 저작권 리스크와 안전한 데모 방향 정리

## 1. 기본 인식

PaperMeister는 논문 PDF를 OCR하고 검색 가능한 corpus로 바꾸는 방향을 가진다.

이 과정에서 가장 중요한 구분은 다음이다.

- 내부 연구/개인 사용용 corpus 구축
- 외부 공개용 데모 corpus 구축

이 둘은 저작권 리스크가 다르다.

## 2. 내부 사용과 외부 공개의 차이

상대적으로 방어가 쉬운 쪽:

- 적법하게 접근한 논문을 내부적으로 OCR
- 검색, 인덱싱, 분석 목적으로 활용
- 원문 전체를 공개 배포하지 않음

더 위험한 쪽:

- 대량 원문 또는 PDF 페이지를 외부에 직접 노출
- 공개 데모에서 원문 자체를 사실상 대체재처럼 보여줌
- 제3자 논문 corpus를 외부 서비스 자산처럼 배포

즉, 내부 corpus 운영과 공개 데모 자산 구축은 분리해서 생각해야 한다.

## 3. 데모에서 피해야 할 것

- 저작권 있는 논문의 PDF 페이지 이미지를 광범위하게 공개
- 긴 원문 텍스트를 대량으로 노출
- 사용자가 원 출처를 거치지 않고 원문 소비를 대체할 수 있게 만드는 것
- 권한이 불분명한 논문 컬렉션을 데모용 공개 자산으로 사용하는 것

## 4. 권장 데모 정책

안전하고 현실적인 데모 방향은 다음과 같다.

- 오픈액세스 논문 위주 corpus 사용
- PDF 페이지 이미지는 직접 노출하지 않음
- 메타데이터, 짧은 snippet, 키워드, 통계, 구조 정보 중심 시연
- 원문이 필요하면 원 출처 링크로 연결

즉, 데모는 “논문 뷰어”가 아니라  
**corpus platform**으로서의 가치를 보여줘야 한다.

## 5. 왜 이 방향이 좋은가

이 접근의 장점:

- 저작권 리스크를 줄인다
- 제품의 본질인 corpus화와 분석 가능성을 더 잘 보여준다
- 단순 PDF viewer로 오해받지 않는다
- 오픈액세스 기반 showcase corpus를 만들기 쉽다

## 6. 데모에서 보여주기 좋은 것

- 검색 질의와 결과 목록
- keyword frequency
- term co-occurrence
- section/caption/body 구분 예시
- corpus 규모와 coverage
- 분야별 용어 분포
- 원문 링크가 포함된 metadata view

## 7. 정책 원칙

PaperMeister 데모는 다음 원칙을 따른다.

1. 가능한 한 OA 논문 중심으로 구성한다
2. 원문 전체나 PDF 페이지 이미지를 기본 데모 대상으로 삼지 않는다
3. snippet은 보수적으로 짧게 유지한다
4. 원문 접근은 출처 링크 중심으로 유도한다
5. 데모는 텍스트 소비보다 corpus 기능을 보여주는 데 초점을 둔다

## 8. 요약

PaperMeister의 데모는 가능하지만, 가장 안전한 방향은:

- 오픈액세스 논문 위주
- 페이지 이미지 비공개
- 키워드/검색/구조/분석 중심 시연

즉, **원문을 보여주는 데모보다 corpus를 보여주는 데모**가 더 적절하다.
