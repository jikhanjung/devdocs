# KOFHIN 매뉴얼 (관리자 + 일반사용자)

[KOFHIN](fsis2026-overview.md) (Korea Fossil Heritage Inventory, 프로덕션 URL `https://fsis.psok.or.kr`) 의 운영·입력 매뉴얼. 2026년 4월 기준 `docs/admin_manual.md` (379줄) + `docs/user_manual_data_entry.md` (303줄) 에 기록된 공식 가이드를 정리한 것이다. 기술 아키텍처는 다른 페이지에서 다루지만, 이 페이지는 **"어떤 화면에서 누가 무엇을 어떻게"** 수행하는지에 초점을 둔다.

## 사용자 그룹과 역할

| 그룹 | 권한 | 주요 업무 |
|---|---|---|
| **일반사용자** | 로그인 후 논문·분류군·표본 입력 | 논문 등록, 분류군·표본 메타데이터 입력 |
| **Professors (관리자)** | 전체 관리 페이지 접근 | 입력 데이터 검토, PDF 파이프라인, 좌표 검토, 화석산지 등록, 위험도 평가, 시스템 관리 |

## 작업 흐름 (하이레벨)

```
일반사용자 입력                     관리자 작업
──────────────────                ──────────────────────────────────
논문 등록 (PDF 업로드)     →      PDF 처리 파이프라인 실행
분류군/표본 정보 입력      →      입력 데이터 검토/확인
표본 위경도 입력           →      추출 좌표 검토 (검토/관리필요/제외)
                                  ↓
                                  화석산지 후보 지도에서 산지 등록
                                  ↓
                                  화석산지 위험도 평가 (Brilha P0~P4)
```

## 상단 네비게이션 (공통)

| 메뉴 | 설명 |
|---|---|
| **KOFHIN** | 메인 페이지 (FSIS 표본 목록) |
| **화석표본** | 표본목록, 표본지도, 변경내역 |
| **화석산지** | 산지목록, 산지지도, 위험도 대시보드, 모식표본 목록 |
| **KPRDB** | 논문목록, 저자목록, 저널목록 |
| **참조** | 지질시대표, 지층정보, 학명정보 |
| **사용자명** | 내 정보, 로그아웃 |
| **시스템관리** (관리자) | 사용자·접속기록·편집기록·PDF·추출·매칭·후보지도 |

## Part A. 일반사용자 — 데이터 입력

### A.1 작업 순서

```
1. 저자 등록 (미등록 저자인 경우)
2. 저널 등록 (미등록 저널인 경우)
3. 논문 등록 (PDF 업로드 포함)
4. 분류군(Taxa) 입력
5. 표본 정보 입력 (표본번호, 위경도 등)
```

### A.2 저자 등록

**KPRDB > 저자목록 > [New]**. 필드: 이름(영문, `성, 이름` 순), 이름(한글, 선택), 소속(선택).

**예**: `Park, Tae-Yoon` / `박태윤` / `한국지질자원연구원`

### A.3 저널 등록

**KPRDB > 저널목록 > [New]**. 필드: 저널명(영문, 필수), 저널명(한글, 선택), 약칭, ISSN.

### A.4 논문 등록

**KPRDB > 논문목록 > [New]**.

#### 기본 정보

| 필드 | 필수 | 예시 |
|---|:---:|---|
| 영문제목 | ✓ | `Late middle Cambrian trilobite faunas...` |
| 한글제목 |  | `조선누층군 세송층 하부의...` |
| 발표년도 | ✓ | `2013` |
| 권 |  | `87` |
| 호 |  | `6` |
| 페이지 |  | `991-1003` |
| 저널 | ✓ | 드롭다운 / 자동완성 |
| 언어 |  | `EN` / `KO` |
| DOI |  | `10.1017/jpa.2013.64` |

#### 분류 정보

| 필드 | 선택값 |
|---|---|
| **문헌종류** | Journal Article / Book / Report / Thesis |
| **분류유형** | 분류학(표본번호 있음) / 분류학(표본번호 없음) / 비분류학 / 해외화석 |
| **화석종류** | 거화석 / 미화석 / 생흔화석 |
| **일반명** | 삼엽충, 이매패류, 복족류, 공룡 등 (복수 선택) |

**분류유형 선택 기준**:

| 유형 | 설명 | 예 |
|---|---|---|
| 분류학(표본번호 있음) | 표본 기재 논문, 표본번호 명시 | 신종 기재, 재기재 |
| 분류학(표본번호 없음) | 분류학적이나 표본번호 미명시 | 화석군 개관, 생층서 |
| 비분류학 | 화석 관련이나 분류학 아님 | 퇴적환경, 고환경 복원 |
| 해외화석 | 한반도 외 지역 | 비교 목적 등록 |

#### 저자 연결

- 논문 등록 폼 하단에 저자 추가 영역
- 자동완성 필드 → 목록 선택
- **저자 순서를 정확히** (1저자, 2저자, ...)
- 드래그앤드롭으로 재정렬 가능 (grip 아이콘)

#### PDF 업로드

- **파일** 필드에서 PDF 선택
- 업로드 후 시스템이 자동으로:
  - 텍스트 레이어 추출 시도
  - OCR (필요 시, 스캔본 대상)
  - 좌표/표본번호 정규식 추출
  - Claude 보강 (cron에 의해 비동기)

### A.5 분류군 (Taxa) 입력

**논문 상세 > [Edit Taxa]**.

| 필드 | 예시 |
|---|---|
| Taxon 이름 (학명, 자동완성) | `Redlichia chinensis` |
| 층 | `Sesong Formation` |
| 시대 | `Cambrian Series 3` |
| 표본사진 수 | `5` (피겨가 있는 표본 수) |
| 기타 표본 수 | `12` (피겨 없는 표본 수) |
| 비고 | 추가 메모 |

**자동완성에 학명 없을 때**: `참조 > 학명정보 > [New]` 에서 학명 + 명명 저자 (`Walcott, 1905` 등) 등록 후 재시도.

### A.6 표본 정보 입력

**논문 상세 > 분류군 테이블 > 연필 아이콘(편집)**.

| 필드 | 예시 |
|---|---|
| 수장기관표본번호 | `KIGAM-F-2024-001` |
| 피겨번호 | `Fig.3-A` |
| 위치(행정구역) | `경상남도 진주시 정촌면` |
| 위도 | `35°07'45''N` 또는 `35.129167` |
| 경도 | `128°06'02''E` 또는 `128.100556` |
| 비고 | `Holotype` |

#### 위경도 입력 규칙

| 형식 | 위도 예 | 경도 예 |
|---|---|---|
| DMS (도분초) | `35°07'45''N` | `128°06'02''E` |
| DMS (소수점) | `35°07'45.5''N` | `128°06'02.3''E` |
| Decimal | `35.129167` | `128.100556` |

파서는 [evaluator.md의 좌표 파서](evaluator.md#gheval-포팅-기능)와 동일 로직을 사용.

### A.7 보조 데이터 (지층, 지질시대)

- `참조 > 지층정보`: 신규 지층 추가 (영문/한글/단위)
- `참조 > 지질시대표`: 사전 등록되어 있음, 일반사용자는 대체로 조작 불필요

### A.8 주의사항

- **중복 확인 필수**: 제목·저자·연도로 검색 후 등록
- **저자 자동완성 활용** — 수동 입력으로 중복 저자 생성 금지
- **PDF 저작권 유의**
- 입력 후 논문 상세에서 전체 검토

## Part B. 관리자 — 운영 워크플로우

### B.1 관리자 메뉴 (시스템관리 드롭다운)

| 메뉴 | URL | 용도 |
|---|---|---|
| 사용자 목록 | `/kprdb/user_list_admin/` | 계정 관리, Professors 그룹 배정 |
| FSIS 접속기록 | `/fsis/all_activity_list_admin/` | FSIS 활동 기록 |
| KPRDB 접속기록 | `/kprdb/all_activity_list_admin/` | KPRDB 활동 기록 |
| 편집기록 | `/kprdb/history_overview/` | 최근 편집 이력 (논문/저널/저자/분류군) |
| **PDF 처리현황** | `/kprdb/pdf_status/` | 텍스트 레이어/OCR/추출 현황 배지 |
| 추출 표본번호 | `/kprdb/extracted_specimens/` | PDF 자동 추출 표본번호 664건 |
| KPRDB 표본목록 | `/kprdb/taxon_specimens/` | 수동 입력 표본과 FSIS 매칭 |
| **화석산지 후보 지도** | `/kprdb/site_candidates/` | 좌표 aggregate 지도 + 산지 등록 |

### B.2 입력 데이터 검토

1. **시스템관리 > 편집기록** — 최근 변경 이력 검토 (변경 전/후 비교)
2. **KPRDB > 논문목록** — 최근 등록 논문 검토:
   - 기본정보 (제목/연도/저널/저자) 정확성
   - 분류유형 적절성
   - 분류군 정보 일치 여부
   - 표본번호 형식, 위경도 정확성

### B.3 표본 매칭 확인 (두 페이지)

| 페이지 | 대상 | 특징 |
|---|---|---|
| **KPRDB 표본목록** (`/kprdb/taxon_specimens/`) | 수동 입력 ReferenceTaxonSpecimen 467건 | FSIS MuseumSpecimen과 표본번호 기준 매칭, [FSIS 매칭] 배지 필터 |
| **추출 표본번호** (`/kprdb/extracted_specimens/`) | PDF 자동 추출 664건 | 기관별 필터(KOPRIF, JBNU, SNU 등), FSIS/KPRDB 양방향 매칭 |

### B.4 PDF 처리 파이프라인 (관리 UI)

**시스템관리 > PDF 처리현황**. 상단 통계 카드:

- **텍스트 레이어**: 있음 / 없음 / 미확인
- **OCR**: 완료 / 실패 / 미처리
- **UID**: 있음 / 없음
- **산지 추출**: 완료 / 미시도 / 데이터없음

배지 클릭 → 해당 상태 논문만 필터.

#### 개별 논문 처리 (상세 페이지 하단)

**논문 처리** 섹션의 [▶] 버튼:

1. **텍스트 레이어 확인**
2. **OCR 처리** (스캔본인 경우, 수 분 소요)
3. **UID 생성**
4. **산지/표본 추출**

또는 **[전체 실행]** 버튼으로 1~4 순차.

#### 일괄 처리 (서버 management command)

대량 처리가 필요하면 서버에서:

```bash
export DATABASE_PATH=/srv/fsis2026/db.sqlite3
export MEDIA_ROOT=/srv/fsis2026/uploads
PYTHON=/home/honestjung/venv/fsis2026/bin/python

$PYTHON manage.py check_text_layers
$PYTHON manage.py run_ocr          # 건당 5~15분
$PYTHON manage.py generate_uids
$PYTHON manage.py extract_pdf_data --all
```

호스트에서 실행 시 WAL 체크포인트 자동 적용. Docker 컨테이너에서 변경이 안 보이면 `docker compose restart`.

OCR 실패는 **PDF 처리현황에서 실패 배지 → 개별 재시도**. 파일 손상·특수 형식은 수동 처리.

→ 기술 상세: [PDF 처리 파이프라인](pdf-pipeline.md), [배포 인프라의 cron](deployment.md)

### B.5 추출 좌표 검토

PDF에서 자동 추출된 좌표는 **반드시** 사람의 검토를 거쳐야 한다.

논문 상세 > **PDF 추출 데이터 > 좌표** 테이블에서 라디오버튼으로 상태 선택:

| 상태 | 의미 | 후속 작업 |
|---|---|---|
| **검토** | 아직 확인 안 함 (기본값) | 확인 필요 |
| **관리필요** | 화석산지로 등록/관리 대상 | 화석산지 후보 지도에서 등록 |
| **제외** | 화석산지 아님 | 후보 지도에서 자동 제외 |
| **등록됨** | 이미 FossilSite로 등록 완료 | 추가 작업 없음 |

#### 검토 기준

**제외** 분류:
- 논문 연구 지역 개관의 일반 지명 좌표
- 해외 비교 표본 좌표
- 지질도/지형도 참조 좌표 (화석 산출 무관)

**관리필요** 분류:
- 화석이 실제로 산출된 지점
- 논문의 locality/section으로 명시된 좌표
- 표본의 산출지 좌표

상태 변경 시 **즉시 자동 저장** (행이 노란색 → 녹색 깜빡임). 변경 이력은 논문의 `extracted_data` JSON에 기록.

### B.6 화석산지 관리

#### 화석산지 후보 지도 (`/kprdb/site_candidates/`)

세 종류 마커:

| 색상 | 의미 |
|---|---|
| **녹색** | 기존 등록된 FossilSite |
| **파란색** | 좌표 그룹이 기존 산지에 연결됨 |
| **주황색** | 미등록 후보 — 검토 후 등록 필요 |

좌측 사이드바: 후보만 / 연결됨 / 전체 필터, 검색, 항목 클릭 → 지도 이동.

#### 후보에서 화석산지 등록

1. 주황색 마커 클릭 → 팝업
2. 위치·좌표·관련 논문 확인
3. **[화석산지 등록]** → 확인 다이얼로그
4. 시스템 자동 처리:
   - FossilSite 생성 (`site_code: FS-YYYY-NNNN` 자동 채번)
   - **Nominatim 역지오코딩**으로 시도/시군구/읍면동 자동 입력
   - 관련 논문과 `FossilSiteReference` 연결
5. 등록 완료 후 버튼 녹색 + 산지코드 표시

#### 화석산지 상세 편집

**화석산지 > 산지목록 > [편집]**에서 추가 정보 입력: 산지명(영문), 상세주소, 지질시대, 암석단위, 암종, 화석군, 화석설명, 학술적 가치, 보호현황, 최초보고연도.

### B.7 위험도 평가 (Brilha → P0~P4)

**화석산지 > 위험도 대시보드**에서 전체 평가 현황.

#### 새 평가 작성

화석산지 상세 > **[새 평가]**. 5개 항목 입력 (자연재해, 인간활동, 법적 보호, 접근성, 토지피복). 시스템이 자동:

- **Overpass API**로 최근접 도로 거리 측정
- **위성영상 기반 토지피복 분석** (PIL+NumPy HSV+ExG)
- 종합 risk_score + P0~P4 자동 산출

상세 수식과 가중치: [GeoHeritage Evaluator](evaluator.md).

평가 이력: 상세 페이지 > **[평가이력]** 에서 시계열 확인.

### B.8 시스템 관리

#### 사용자 관리

`/kprdb/user_list_admin/` — 추가/편집/비밀번호 변경/삭제. **Professors 그룹 배정 → 관리자 권한**.

#### 접속 기록

- FSIS 접속기록: 화석표본/산지 페이지 활동
- KPRDB 접속기록: 논문/저자/저널 페이지 활동

#### 데이터 백업

- DB: `/srv/fsis2026/db.sqlite3` (자동 cron)
