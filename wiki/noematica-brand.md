# Noematica — 회사 브랜드 & Naming

**Noematica** (노에마티카)는 이 위키가 다루는 프로젝트들의 상위 회사 브랜드다. 2026-04-03의 `docs/naming/` 세션에서 광범위한 네이밍 탐색 후 확정되었으며, PaperMeister·SCODA·PaleoBase·Trilobase를 모두 포괄하는 **scholarly infrastructure / knowledge systems 회사**로 포지셔닝한다.

이 페이지는 현재 PaperMeister 섹션 하위에 배치되어 있지만, 내용은 **Noematica 전체**를 다룬다. 향후 다른 프로젝트들도 Noematica 브랜드 하에 정리될 때 "Meta / Brand" 최상위 섹션으로 이동할 수 있다.

## 1. 최종 결정

**회사명**: **Noematica**
**한국어**: **노에마티카**

### 왜 선택되었나

`Noematica`는 다음 기준에서 가장 잘 균형을 이룬 후보:

- 개념적 깊이 (`noema` = 의미 있는 개념적 내용, 사유의 대상)
- 학술적 적합성
- 인프라 호환성 (제품·플랫폼·패키지를 모두 담을 수 있음)
- 장기 확장성
- 한국어 발음 자연성

### 회피한 프레임

| 회피 카테고리 | 이유 |
|---|---|
| 고생물학 특화 (`Paleo...`) | 회사 범위를 너무 일찍 한 도메인에 lock-in |
| 학술 보수 (`Scholar...`, `ScholarForge`) | 의미는 강하지만 톤이 전통적·구식 |
| NLP/텍스트 처리 (`Corpus...`, `CorpusForge`) | 기술적으로 정확하지만 범위가 좁게 해석됨 |
| 법률/의학 (`Evidence...`, `Evidentia`) | 철학은 맞지만 톤이 forensic/legal 쪽 |
| 지질학 (`Strata...`, `Strataform`) | 구조/레이어 개념은 좋지만 geological association이 남음 |

### Final Shortlist (Final Direction Memo)

결정 단계의 Top 3:

| 후보 | 강점 | 리스크 | Best if... |
|---|---|---|---|
| **NoemaEngine** | 시스템/플랫폼 정체성, 안정적, 한국어 `노에마엔진` 자연스러움 | 약간 기능적·기술적 톤, 제품/엔진처럼 들릴 수 있음 | **Stability + infrastructure identity** 우선 |
| **NoemaForge** | 개념적 깊이 + 만드는 느낌, ScholarForge보다 현대적 | 더 추상적, `Forge`는 기술 외 독자에게 설명 필요 | **Brand character + 빌더 에너지** 우선 |
| **NoosForge** | 가장 sharp한 현대 브랜드 느낌, 컴팩트 | `Noos`는 `Noema`보다 덜 즉시 해석됨, 한국어 rendering 불안정 (`누스포지` / `노오스포지`) | **Compact + modern brand sharpness** 우선 |

**보조 후보**: `Noemata` (노에마타) — 우아하지만 infrastructural 느낌 약함

### 최종 선정 이유

`NoemaEngine`/`NoemaForge`/`NoosForge` 사이에서 결국 **`Noematica`**가 선택된 것은 Brand Package 문서에서다. `Noematica`는:
- `Noema` 루트의 개념적 강점을 유지
- `-tica` 접미로 완전한 회사명 형태 (company name으로 더 natural)
- 한국어 `노에마티카`가 가장 안정적 (다른 후보보다 덜 flex)
- Engine/Forge 같은 접미 없이 **자체로 완결된 이름**
- 도메인 확보 가능성(`noematica.ai`, `noemati.ca`)이 브랜드 느낌 강화

## 3. Core Brand Meaning

`Noematica`는 `noema`를 뿌리로 한다:

- 의미 있는 개념적 내용
- 사유의 내용
- 의미 지니는 단위
- thought-object

브랜드적으로 `Noematica`는 다음에 관심 있는 회사로 이해된다:

- 의미 있는 지식 구조
- 학술 시스템
- 연구 코퍼스 인프라
- 재사용 가능한 지식 artifact

이 범위는 다음을 모두 담기에 충분히 넓다:
- [PaperMeister](papermeister-overview.md)
- [SCODA](scoda.md) (spec) + [SCODA Engine](scoda-engine.md) (runtime)
- [PaleoBase / Trilobase](trilobase-overview.md) + [Packages](packages.md)
- 미래 도메인 패키지

## 4. 도메인 스택

| 도메인 | 역할 | 강점 | 리스크 |
|---|---|---|---|
| **`noematica.ai`** | 메인 글로벌 사이트, 영문 회사 URL | 직관적, 풀 브랜드 보존, 국제 표현에 적합 | `.ai`는 흔해서 "AI company"로 과도하게 코딩될 수 있음 |
| **`noematica.kr`** | 한국어 사이트 또는 랜딩 경로 | 국내 신뢰·접근성, 로컬 아웃리치·기관 신뢰, 한국어 마케팅 지원 | — |
| **`noemati.ca`** | 짧은 브랜드 URL, redirect, 명함/캠페인 | 시각적으로 기억하기 쉬움, elegant domain hack | 즉시 연결이 어려울 수 있음, 일부는 `.ca`를 캐나다 도메인으로 읽음 |

**권장 구조**:
- `noematica.ai` = 메인 회사 사이트
- `noematica.kr` = 한국 페이싱 사이트 또는 redirect
- `noemati.ca` = 짧은 링크 / 브랜드 redirect

## 5. Brand Architecture

```
┌─────────────────────────────────────────────┐
│              NOEMATICA (Company)             │
│   scholarly infrastructure /                 │
│   knowledge systems                          │
└──────────────┬──────────────────────────────┘
               │
       ┌───────┴────────┬────────────┐
       ▼                ▼            ▼
┌────────────┐  ┌─────────────┐  ┌─────────────┐
│PaperMeister│  │    SCODA    │  │ (future     │
│            │  │(spec +      │  │  products)  │
│ corpus &   │  │ engine +    │  │             │
│ literature │  │ hub)        │  │             │
└────────────┘  └──────┬──────┘  └─────────────┘
                       │
              ┌────────┴─────────┐
              ▼                  ▼
        ┌──────────┐      ┌──────────────┐
        │PaleoBase │      │Future Domain │
        │Trilobase │      │  Packages    │
        │(+ others)│      │              │
        └──────────┘      └──────────────┘
                Domain Knowledge Layer
```

### 계층의 의미

- **회사 (Noematica)**: 넓게 유지 — 어떤 도메인에도 구속되지 않음
- **제품 / 플랫폼 (PaperMeister, SCODA)**: 각자 개별 recognizable, 독립적 버전 관리 및 배포
- **도메인 지식 레이어 (PaleoBase, Trilobase, ...)**: 부모 브랜드를 재구성하지 않고 확장 가능

이 구조는 `Paleo...` 계열 이름이 회사 이름에 어울리지 않는 이유이기도 하다 — `Paleo`는 도메인 패키지 네이밍으로 적합하지만, 부모 회사에는 너무 좁다.

## 6. 한국어 Rendering

| 영문 | 한글 | 평가 |
|---|---|---|
| **Noematica** | **노에마티카** | 자연스러움, 발음 안정적, 브랜드명으로 기능 |
| NoemaEngine | 노에마엔진 | 분명함, 실용·안정적 |
| NoemaForge | 노에마포지 | 기억에 남지만 더 이국적 |
| NoosForge | 누스포지 / 노오스포지 | 불안정, rendering 다양 |
| Noemata | 노에마타 | 고유명사로 매끄러움 |

→ `Noema...` 계열이 `Noos...` 계열보다 한국어에서 더 쉽다. `NoosForge`는 영문 브랜드 느낌이 가장 sharp하지만 한국어 안정성이 약해 탈락 원인이 됨.

## 7. Intro 문장 (Brand Package)

### 영문

> Noematica builds scholarly infrastructure for turning documents, corpora, and evidence into reusable knowledge systems.

> Noematica develops research infrastructure for searchable corpora, structured knowledge, and distributable scholarly artifacts.

> Noematica is a scholarly engineering company building the systems behind living research corpora and evidence-based knowledge platforms.

### 한국어

> Noematica는 문헌, 코퍼스, 근거를 재사용 가능한 지식 시스템으로 바꾸는 학술 인프라를 만듭니다.

> Noematica는 검색 가능한 연구 코퍼스와 구조화된 지식 아티팩트를 위한 scholarly engineering 회사입니다.

> Noematica는 흩어진 문헌을 살아 있는 연구 코퍼스와 지식 시스템으로 전환하는 인프라를 구축합니다.

- 문헌을 살아 있는 지식 인프라로
- 논문을 재사용 가능한 지식으로
- 연구 코퍼스와 지식 시스템을 위한 인프라
- 문헌을 지식 자산으로 바꾸는 기반

## 9. Brand Feel Summary

`Noematica`는 다음처럼 느껴진다:

- 지적 (intellectual)
- 현대적 (modern)
- 인프라 호환적 (infrastructure-compatible)
- 약간 추상적이지만 공허하지 않음
- 단일 제품 레이블보다는 **진지한 장기 회사**에 가깝다

가장 직접적인 이름은 아니지만, 이것이 장점이 될 수 있다 — 회사가 더 넓은 정체성으로 성장할 때 이름이 구속이 되지 않는다.

## 10. Risks to Watch

브랜드 선호가 있더라도 다음은 여전히 확인 필요:

- 도메인 가용성 (`noematica.ai`, `noematica.kr`, `noemati.ca`)
- 상표 충돌 screening
- 주요 관할 지역의 회사명 검색
- GitHub / 소셜 핸들 가용성
- 비슷한 발음 이름과의 혼동

## 11. 다음 단계 (Brand Package의 추천)

1. 가용하면 **코어 도메인 먼저 확보**
2. 실제 사용 시나리오로 이름 테스트:
   - 웹사이트 헤더
   - 회사 소개 문장
   - 이메일 서명
   - 명함
   - 한국어·영어 대화 테스트
3. 전면 commitment 전에 네이밍 충돌 최종 점검

## 12. 위키에서의 위치

이 페이지는 현재 **PaperMeister 섹션 하위**에 있지만, 다루는 범위는 Noematica 전체다. 이유:

- Naming/brand 문서가 `docs/naming/` (PaperMeister 레포지토리) 안에 있었다
- 이 devdocs 위키에는 아직 top-level "Brand / Meta" 섹션이 없다
- 미래에 SCODA/Trilobase 페이지들도 Noematica 브랜드를 명시적으로 참조하게 되면, 이 페이지를 최상위로 올리는 것이 자연스러워진다

당장은 위키 전체에서 이 페이지 하나만이 Noematica 브랜드를 다루며, 다른 관련 페이지들([SCODA 아키텍처](scoda.md), [Trilobase 개요](trilobase-overview.md) 등)에서 "Noematica 생태계의 일부" 형태로 가벼운 back-link을 걸 수 있다.
